# 项目规则

## 环境信息

### Python 环境
- Python 版本：3.12
- Python 安装路径：`D:\0Code2\py312`

### 代码执行规则
- 主入口文件：`main.py`
- 执行命令（Windows CMD环境）：`cmd /c main.bat`
- 执行命令（Windows PowerShell环境）：`powershell -Command "Invoke-Expression -Command 'cmd.exe /c main.bat'"`

说明：
- 所有 Python 代码运行时必须使用 `main.py` 作为主入口
- Windows环境下使用 `main.bat` 批处理文件执行
- 设置 `PYTHONIOENCODING=utf-8` 确保输出编码正确

## 文件处理规则

### 临时文件管理
- **临时区目录**：所有临时文档、代码、文件必须放置在 `tmp` 目录下
- **清理规则**：完成任务后必须删除临时文件，但保留 `tmp` 目录本身
- **使用场景**：中间处理文件、缓存文件、测试文件等

### 数据库文件管理
- **数据库目录**：所有数据库文件必须保存在 `db` 目录下
- **支持格式**：
  - SQLite3 数据库文件（推荐用于复杂查询）
  - CSV 文件（推荐用于简单表格数据）
  - JSON 文件（推荐用于键值对或配置数据）
- **选择建议**：根据数据复杂度和查询需求选择合适的格式

### 配置文件管理
- **配置目录**：所有配置文件必须保存在 `config` 目录下
- **支持格式**：
  - CSV 文件（推荐用于表格配置）
  - JSON 文件（推荐用于键值对配置）
- **配置说明**：
  - 敏感信息（如Webhook地址）应使用配置文件管理，不应硬编码
  - 配置文件示例：`config/FeiShu.csv`（飞书机器人配置）

### 日志系统规则
- **日志目录**：所有日志文件必须保存在 `log` 目录下
- **文件命名格式**：`YYYYMMDD.log`（例如：`20260331.log`）
- **日志保留策略**：
  - 默认保留最近 30 天的日志文件
  - 超过 30 天的日志文件应自动清理
  - 如需特殊保留期限，需在项目文档中明确说明
- **日志级别**：建议包含 DEBUG、INFO、WARNING、ERROR、CRITICAL 等级别
- **日志内容**：应包含时间戳、日志级别、模块名、日志消息等关键信息

## 版本管理规则

### 文档管理
- **项目说明书**：使用 `README.md` 作为项目的说明书及版本迭代记录
- **语言规范**：默认使用简体中文（zh-CN）编写文档
- **文档内容**：应包含项目介绍、功能说明、使用方法、版本更新记录等内容

### 版本号规则
- **版本号格式**：`YY.M.DX`（例如：`26.3.31A`）
  - `YY`：年份后两位（如 26 代表 2026 年）
  - `M`：月份（如 3 代表 3 月）
  - `D`：日期（如 31 代表 31 日）
  - `X`：当天版本标识，使用大写字母 A-Z 顺序递增
- **版本升级条件**：
  - 项目产生重大变更时必须增加版本号
  - 同一天内的多次变更使用 A、B、C...Z 区分
  - 跨天后版本号日期部分更新，字母重置为 A
- **版本记录**：每次版本更新必须在 `README.md` 中记录变更内容

## 隐私处理规则

### 敏感信息保护
- **禁止明文信息**：项目中代码以及文档禁止出现明文敏感信息，包括但不限于：
  - IP 地址
  - Token 值
  - API 密钥
  - 密码值
  - 其他敏感凭证
- **文档说明**：如果文档中需要说明敏感信息，必须使用 `***` 进行隐写处理
- **代码示例**：代码示例中的敏感信息也应使用 `***` 替代
- **配置文件**：配置文件中的敏感信息应使用环境变量或配置管理工具管理，不应硬编码

### 隐写示例
- IP 地址：`192.168.1.***` 或 `***.***.***.***`
- Token 值：`abc123***` 或 `***`
- API 密钥：`sk-***` 或 `***`
- 密码值：`password***` 或 `***`
